# Stacks Plugins

Stacks Plugins is a collection of **AI agent integrations** for the [Stacks](https://www.stacks.co/) blockchain. Each plugin exposes the same **19 core tools** — balances, STX transfers, account history, PoX stacking, BNS naming, Clarity contract calls, ALEX swaps, and Allbridge bridging — across three agent frameworks.

Tool logic lives in **`@stacks/agent-core`** (`plugins/stacks/agent-core`). Framework adapters import the published npm package **`@sugarhi11/agent-core`**, which is built from that source.

## Repository layout

```
stacks-plugins/
├── docs/                       This documentation site (Mintlify)
├── plugins/stacks/
│   ├── agent-core/             @stacks/agent-core — shared tool implementations
│   ├── eliza/                  plugin-eliza-stacks
│   ├── openclaw/               @stacks/openclaw-stacks
│   └── hermes/                 Hermes Python plugin + Node bridge
└── README.md
```

There is no root npm workspace. Each package under `plugins/stacks/` has its own `package.json` and is built independently.

## What's included

| Package | Path | Role |
| --- | --- | --- |
| `@stacks/agent-core` | `plugins/stacks/agent-core` | Framework-agnostic tool implementations (local source) |
| `@sugarhi11/agent-core` | npm | Published package consumed by framework adapters |
| `plugin-eliza-stacks` | `plugins/stacks/eliza` | ElizaOS actions + `STACKS_WALLET` provider |
| `@stacks/openclaw-stacks` | `plugins/stacks/openclaw` | OpenClaw tool plugin (19 core tools + `stacks_wallet_info`) |
| Hermes plugin | `plugins/stacks/hermes` | Python registration, hooks, `/stacks` command, bundled skills |

## Design principles

<AccordionGroup>
  <Accordion title="Single source of truth">
    The `STACKS_TOOLS` registry in agent-core defines stable tool names, descriptions, and handlers. ElizaOS and OpenClaw import handlers from `@sugarhi11/agent-core`; Hermes routes all calls through `stacks-bridge.mjs` to the same handlers.
  </Accordion>
  <Accordion title="Read vs write separation">
    Read tools query the [Hiro API](https://docs.hiro.so/stacks/api). Write tools sign and broadcast transactions via `@stacks/transactions` and require an explicit `senderKey` (or `STACKS_SENDER_KEY` from the environment).
  </Accordion>
  <Accordion title="Network-aware">
    Every tool accepts an optional `network` parameter (`mainnet` or `testnet`). Defaults vary by framework — see [Configuration](/configuration).
  </Accordion>
</AccordionGroup>

## Requirements

- **Node.js** 22+ (TypeScript plugins and Hermes Node bridge)
- **Python** 3.9+ (Hermes plugin — no required pip packages)
- A Hiro API endpoint (public mainnet/testnet URLs are used by default)

<Note>
  Write tools move real funds on-chain. Always test on **testnet** first and never expose private keys in prompts or logs.
</Note>
