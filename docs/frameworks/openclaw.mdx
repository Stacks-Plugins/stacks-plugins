# OpenClaw plugin

`@stacks/openclaw-stacks` registers Stacks blockchain tools with OpenClaw's tool plugin API, using TypeBox schemas for parameter validation.

## Installation

```bash
cd plugins/stacks/openclaw
npm install
npm run build
```

Install into OpenClaw from the local path:

```bash
openclaw plugins install ./plugins/stacks/openclaw
```

Or install from a published package once available.

Validate the plugin bundle:

```bash
npm run plugin:validate
```

## Tools

OpenClaw exposes **34 tools**:

- **33 core tools** from `@sugarhi11/agent-core` (`STACKS_TOOLS`), including sBTC and Zest
- **`stacks_wallet_info`** — shows configured wallet address, network, and signing status (OpenClaw-specific)

## Plugin manifest

OpenClaw reads `openclaw.plugin.json` for metadata, tool contracts, and configuration:

```json
{
  "id": "stacks",
  "name": "Stacks Plugin",
  "configSchema": {
    "properties": {
      "defaultNetwork": { "enum": ["mainnet", "testnet"] }
    }
  },
  "contracts": {
    "tools": [
      "stacks_wallet_info",
      "stacks_get_balance",
      "stacks_send_tokens"
    ]
  }
}
```

## Write tools are optional

Tools that sign transactions are registered with `{ optional: true }` in `toolMetadata`:

- `stacks_send_tokens`
- `stacks_stack`
- `stacks_delegate_stx`
- `stacks_revoke_delegate`
- `stacks_transfer_name`
- `stacks_contract_call`
- `stacks_swap_execute`
- `stacks_bridge_initiate`
- `stacks_send_sbtc`
- `stacks_sbtc_initiate_peg_in`
- `stacks_sbtc_initiate_peg_out`
- `stacks_zest_supply_sbtc`
- `stacks_zest_redeem_sbtc`
- `stacks_zest_collateral_add_sbtc`
- `stacks_zest_borrow`
- `stacks_zest_repay`

This lets agents run in **read-only mode** by default. Enable write tools only when the agent has authorization to sign.

## Configuration

Set the default network in plugin config:

```json
{
  "defaultNetwork": "testnet"
}
```

Individual tools still accept a `network` parameter to override per call.

## Tool response format

Each tool's `execute` handler returns formatted text from agent-core handlers. Errors throw with a descriptive message.

## Tests
