# Configuration

Configure network targeting and transaction signing through environment variables and per-tool parameters.

## Environment variables

| Variable | Required | Default | Description |
| --- | --- | --- | --- |
| `STACKS_NETWORK` | No | See below | Target network: `mainnet` or `testnet` |
| `STACKS_SENDER_KEY` | For write tools | — | Hex-encoded private key used to sign transactions |
| `STACKS_WALLET_ADDRESS` | No | Derived from sender key on active network | Explicit agent wallet address when sender key is set |
| `BITCOIN_PRIVATE_KEY` | Peg-in only | — | WIF or hex Bitcoin key for sBTC peg-in |
| `BITCOIN_ADDRESS` | Peg-in only | — | Bitcoin address holding UTXOs for peg-in |

Create your local env file from the example:

```bash
cp plugins/stacks/.env.example plugins/stacks/.env
```

Example `.env` for testnet development:

```bash
STACKS_NETWORK=testnet
STACKS_SENDER_KEY=e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b85501
# BITCOIN_PRIVATE_KEY=...   # peg-in only (WIF or hex)
# BITCOIN_ADDRESS=tb1...      # peg-in only
```

To derive Leather-compatible keys locally without committing secrets:

```bash
# From repo root (PowerShell example)
$env:MNEMONIC="your 24 words"; $env:STACKS_NETWORK="testnet"
node scripts/derive-leather-env.mjs
```

Copy the printed values into `plugins/stacks/.env` or `~/.hermes/.env`.

For Hermes, set these in `~/.hermes/.env`.

<Warning>
  `STACKS_SENDER_KEY` grants full control of the associated wallet. Use a dedicated testnet wallet for development. Never commit this value or pass it through untrusted agent prompts.
</Warning>

## Per-tool network override

Every tool accepts an optional `network` parameter that overrides the default:

```json
{
  "address": "ST1PQHQKV0RJXZFY1DGX8MNSNYVE3VGZJSRTPGZGM",
  "network": "testnet"
}
```

| Framework | Default when omitted |
| --- | --- |
| `@sugarhi11/agent-core` handlers | `mainnet` |
| ElizaOS | `testnet` (via `STACKS_NETWORK` env, then fallback) |
| OpenClaw | `mainnet` (configurable via `defaultNetwork` in plugin config) |
| Hermes | `STACKS_NETWORK` env var injected by bridge, then handler default |

## OpenClaw plugin config

The OpenClaw manifest (`openclaw.plugin.json`) supports a `defaultNetwork` setting via `configSchema`:

```json
{
  "defaultNetwork": "testnet"
}
```

Write tools are registered as **optional** in OpenClaw. Enable them only when your agent is authorized to sign transactions.

## Hermes requirements

The Hermes plugin (`plugin.yaml`) documents that write tools require `STACKS_SENDER_KEY`. Read-only tools work without a sender key; write tools are gated by a `check_fn` when signing is unavailable.

**Runtime requirements:**

- Node.js 22+ on `PATH`
- `npm install` in `plugins/stacks/hermes/` (installs `@sugarhi11/agent-core` for the bridge)
- `STACKS_NETWORK` and `STACKS_SENDER_KEY` in `~/.hermes/.env` (or the process environment)

The Node bridge runs with `cwd` set to `plugins/stacks/hermes/` and loads `@sugarhi11/agent-core` from `hermes/node_modules`.

For write tools, the bridge injects `STACKS_SENDER_KEY` into params when `senderKey` is not provided explicitly.

## Hiro API endpoints

| Network | Base URL |
| --- | --- |
| Mainnet | `https://api.hiro.so` |
| Testnet | `https://api.testnet.hiro.so` |

No API key is required for public Hiro endpoints at default rate limits. For production agents with high volume, consider a [Hiro API key](https://docs.hiro.so/stacks/api).

## External APIs

| Service | Used by | Endpoint |
| --- | --- | --- |
| ALEX DEX | `stacks_swap_quote`, `stacks_swap_execute` | `https://api.alexgo.io` |
| Allbridge Core | `stacks_bridge_quote`, `stacks_bridge_initiate` | `https://core.api.allbridgecoreapi.net` |
| Emily / sBTC API | `stacks_sbtc_*` | Network-specific sBTC Emily endpoints via `sbtc` SDK |
| Zest v2 contracts | `stacks_zest_*` | On-chain read/call via Hiro API |

Swap and bridge tools are primarily oriented toward **mainnet** liquidity.

## Signing parameters

Write tools accept these common signing fields:

| Parameter | Type | Description |
| --- | --- | --- |
| `senderKey` | `string` | Hex-encoded private key (required for writes) |
| `fee` | `string \| number` | Transaction fee in microSTX (auto-estimated if omitted) |
| `nonce` | `string \| number` | Account nonce (auto-fetched if omitted) |

<Tip>
  Let the SDK estimate `fee` and `nonce` during development. Override them only when you need deterministic transaction ordering or custom fee rates.
</Tip>

## Wallet aliases

Eliza and Hermes resolve friendly address aliases (`"my wallet"`, `"me"`, `"agent wallet"`) to the configured agent wallet address. When `STACKS_SENDER_KEY` is set, the address is **derived for the active network** (`mainnet` vs `testnet`) unless `STACKS_WALLET_ADDRESS` overrides it.
