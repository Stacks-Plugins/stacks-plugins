# Tool overview

Every framework plugin exposes the same **33 core tools** defined in agent-core's `STACKS_TOOLS` registry. OpenClaw additionally registers **`stacks_wallet_info`** for wallet/network status.

## Read vs write

<CardGroup cols={2}>
  <Card title="Read tools (17)" icon="magnifying-glass">
    Query chain state via Hiro API, Emily, or external quote APIs. No private key required.
  </Card>
  <Card title="Write tools (16)" icon="pen-to-square">
    Sign and broadcast transactions. Require `senderKey` (or Bitcoin env for peg-in).
  </Card>
</CardGroup>

## Common parameters

All tools accept:

<ParamField body="network" type="string">
  `mainnet` or `testnet`. Defaults to `mainnet`.
</ParamField>

Write tools additionally accept:

<ParamField body="senderKey" type="string" required>
  Hex-encoded private key for the signing wallet.
</ParamField>

<ParamField body="fee" type="string | number">
  Transaction fee in microSTX. Auto-estimated when omitted.
</ParamField>

<ParamField body="nonce" type="string | number">
  Account nonce. Auto-fetched when omitted.
</ParamField>

## Response shapes

**Read tools** return typed JSON objects (see category pages for fields).

**Write tools** return a broadcast result:

```json
{
  "txid": "0x...",
  "success": true
}
```

On failure:

```json
{
  "txid": "",
  "success": false,
  "error": "Not enough funds",
  "reason": "..."
}
```

## Amounts

Unless noted otherwise, STX amounts are in **microSTX** (1 STX = 1,000,000 microSTX).

Fungible token amounts use the token's smallest unit as defined by its contract.
